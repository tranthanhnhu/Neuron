# DSL specification (`snn_mc`)

Technical reference for the line-based DSL parsed by `snn_mc/dsl/parser.py`.
For a runnable tutorial see [huong_dan_su_dung.md](huong_dan_su_dung.md).

## Parser implementation

The parser is a **hand-written line scanner** (split on whitespace, one statement per line).
It is **not** generated by ANTLR/Xtext/Bison today. Typos in block kinds produce
`ValueError` with **did-you-mean** hints (`difflib`). A grammar generator may be
adopted later if the language grows (fork/join, richer expressions).

## Syntax (core statements)

| Line | Meaning |
|------|---------|
| `include <file>` | Inline another DSL file |
| `horizon <int>` | Simulation length; NuSMV clock `t` ranges `0..horizon` |
| `input <name>` | External Boolean stimulus |
| `schedule <input> values TRUE FALSE ...` | Fixed prefix of input trace; tail is nondeterministic |
| `network_output <neuron>` | Declare a network-level output port (observable neuron) |
| `neuron_params <name> tau=... w_exc=... w_inh=... S=... R=... Pmax=...` | LIF parameter bundle |
| `block <kind> key=value ...` | Archetype macro (see below) |
| `edge <src> -> <dst> weight <int>` | Manual synapse |
| `exc` / `inh` | Shorthand edges |
| `spec CTLSPEC\|LTLSPEC ...` | User property |

Block keys are **case-insensitive** (`Input=stim` = `input=stim`).

### `block simple_series` keys

| Key | Required | Description |
|-----|----------|-------------|
| `input` | yes | DSL `input` name or upstream neuron |
| `output` | no | Output neuron (default: last in chain) |
| `N`, `prefix` **or** `neurons` | yes | Chain length / names |
| `weights` | no | Comma list or `[a,b,c]`; one weight per excitatory edge (`input→n1`, `n1→n2`, …) |
| `threshold` | no | Override firing threshold τ for neurons in this block |
| `params` | no | `neuron_params` set name (default `default`) |

### `block negative_loop` keys

| Key | Required | Description |
|-----|----------|-------------|
| `input` | yes | Driver for first neuron |
| `output` | no | Observable output neuron (default: last in loop) |
| `A`, `B` **or** `N`/`prefix` **or** `neurons` | yes | Loop neurons |
| `exc_weights` / `weights` | no | Excitatory edge weights (stim→head, chain, …) |
| `inh_weight` | no | Magnitude of inhibitory back-edge (stored negative) |
| `threshold`, `params` | no | As above |

Equivalent intent to the supervisor’s illustrative form:

```text
block simple_series input=stim output=c4 N=4 prefix=c weights=3,2,5,4 threshold=4 params=default
```

## Semantics

### Weight

Integer **synaptic contribution** added to a neuron’s excitatory/inhibitory input sum when
the source is active (`TRUE` input or `.spike`). It is **not** an SDF token rate and does
not mean “run the receiver N times.”

### Schedule `TRUE` / `FALSE`

At discrete step `t`, for a declared `input`:

- `TRUE` — external stimulation present (contributes via `w_exc` on edges from that input).
- `FALSE` — no stimulation from that input; LIF neurons may still **leak** (membrane evolves).

After the listed prefix, each input is **nondeterministic** `{TRUE, FALSE}` for exploration.

### Clock `t` / `horizon`

`horizon` sets `VAR t : 0..horizon` in NuSMV. Variable `t` is the **global simulation step**,
not threshold τ and not BMC depth. It drives schedules and LIF time steps.

## Expressiveness

| Supported | Not yet in DSL |
|-----------|----------------|
| Seven archetypes via `block` | Fork / join (StreamIt-style) |
| Chaining blocks (`input=c4` from prior block) | Synaptic delay lines |
| Per-edge weights on blocks | Multiple `input=` ports on one block line |
| `horizon`, `network_output` | Compositional assume-guarantee proofs |
| Manual `edge` / `exc` / `inh` | SDF actor rates |
| CLI `--override N=` | |
